## Application Deployment

Dokploy supports multiple application deployment strategies to fit your workflow.

### Source Types

Deploy applications from various sources:

<Accordion title="Git Repositories">
  Connect to any Git provider for automatic deployments:
  
  - **GitHub**: OAuth integration with automatic webhook setup
  - **GitLab**: Support for self-hosted and GitLab.com
  - **Bitbucket**: Cloud and self-hosted Bitbucket
  - **Gitea**: Self-hosted Git service integration
  - **Generic Git**: Any Git repository via SSH or HTTPS
  
  ```typescript
  // Supported source types
  sourceType: "github" | "gitlab" | "bitbucket" | "gitea" | "git"
  ```
  
  Features include:
  - Automatic deployments on push
  - Branch selection and switching
  - Submodule support
  - Custom SSH key authentication
  - Watch paths for selective deployments
</Accordion>

<Accordion title="Docker Registries">
  Pull and deploy pre-built images from:
  
  - Docker Hub
  - GitHub Container Registry (ghcr.io)
  - GitLab Container Registry
  - Private registries with authentication
  - Custom registry URLs
  
  ```typescript
  sourceType: "docker"
  dockerImage: "nginx:latest"
  ```
</Accordion>

<Accordion title="Drop Deployment">
  Upload and deploy applications directly:
  
  - Drag-and-drop file upload
  - Archive extraction (zip, tar.gz)
  - Direct deployment without Git
  
  ```typescript
  sourceType: "drop"
  ```
</Accordion>

### Application Configuration

Fine-tune your deployments with advanced options:

<Tabs>
  <Tab title="Environment Variables">
    Manage environment variables securely:
    
    ```bash
    # Set environment variables
    NODE_ENV=production
    DATABASE_URL=postgresql://user:pass@host:5432/db
    API_KEY=secret-key
    
    # Variables are injected at runtime
    # Support for multi-line values
    # Automatic escaping and validation
    ```
  </Tab>
  
  <Tab title="Resource Limits">
    Control CPU and memory allocation:
    
    ```typescript
    {
      cpuLimit: 2.0,           // Maximum CPU cores
      cpuReservation: 0.5,     // Guaranteed CPU
      memoryLimit: 2048,       // Maximum memory (MB)
      memoryReservation: 512   // Guaranteed memory (MB)
    }
    ```
  </Tab>
  
  <Tab title="Build Configuration">
    Customize the build process:
    
    ```bash
    # Custom build arguments
    --build-arg NODE_VERSION=20
    --build-arg BUILD_ENV=production
    
    # Build secrets (not in image)
    --secret id=npmrc,src=.npmrc
    
    # Custom build path
    dockerFilePath: "./docker/Dockerfile.prod"
    ```
  </Tab>
  
  <Tab title="Command & Args">
    Override container command and arguments:
    
    ```typescript
    {
      command: "node",
      args: ["--max-old-space-size=4096", "server.js"]
    }
    ```
  </Tab>
</Tabs>

### Database Features

<Accordion title="Automated Backups">
  Schedule automatic backups to external storage:
  
  ```typescript
  {
    schedule: "0 2 * * *",  // Daily at 2 AM
    enabled: true,
    database: "myapp",
    prefix: "backup",
    destinationId: "s3-destination",
    keepLatestCount: 7      // Retain 7 backups
  }
  ```
  
  - **Destinations**: S3, MinIO, local storage, any S3-compatible service
  - **Retention**: Automatic cleanup of old backups
  - **Notifications**: Alerts on backup success/failure
  - **Scheduling**: Flexible cron expressions
</Accordion>

<Accordion title="Volume Persistence">
  Databases use Docker volumes for data persistence:
  
  ```typescript
  // Automatic volume creation
  mounts: [{
    type: "volume",
    volumeName: `${appName}-data`,
    mountPath: "/var/lib/postgresql/data"
  }]
  ```
  
  - Survives container restarts
  - Can be backed up
  - Migratable across servers
</Accordion>

<Accordion title="Connection Management">
  Easy access to connection details:
  
  ```bash
  # Internal connection (from applications)
  DATABASE_URL=postgresql://user:pass@postgres-appname:5432/dbname
  
  # External connection (from host)
  Host: your-server-ip
  Port: exposed-port
  Database: dbname
  User: user
  Password: pass
  ```
</Accordion>

## Docker Compose Support

Deploy complex multi-container applications with full Docker Compose compatibility.

### Compose Deployment

<Tabs>
  <Tab title="Repository Compose">
    Deploy compose files from Git repositories:
    
    ```yaml
    # docker-compose.yml
    version: '3.8'
    services:
      web:
        image: nginx:alpine
        ports:
          - "80:80"
      app:
        build: .
        environment:
          - DATABASE_URL=${DATABASE_URL}
      db:
        image: postgres:16
        volumes:
          - pgdata:/var/lib/postgresql/data
    
    volumes:
      pgdata:
    ```
    
    ```typescript
    {
      sourceType: "github",
      repository: "myorg/myapp",
      branch: "main",
      composePath: "./docker-compose.yml"
    }
    ```
  </Tab>
  
  <Tab title="Raw Compose">
    Paste compose content directly:
    
    ```typescript
    {
      sourceType: "raw",
      composeFile: `version: '3.8'
    services:
      app:
        image: myapp:latest
        ports:
          - "3000:3000"`
    }
    ```
  </Tab>
  
  <Tab title="Docker Stack">
    Deploy as Docker Swarm stack:
    
    ```typescript
    {
      composeType: "stack",  // Instead of "docker-compose"
      // Enables swarm-specific features
    }
    ```
  </Tab>
</Tabs>

## Domain Management & Routing

Dokploy integrates with Traefik for automatic routing and SSL certificates.

### Domain Configuration

<Steps>
  <Step title="Add Domain">
    Configure custom domains for your applications:
    
    ```typescript
    {
      host: "app.example.com",
      port: 3000,
      https: true,
      path: "/",
      certificateType: "letsencrypt"
    }
    ```
  </Step>
  
  <Step title="SSL Certificates">
    Automatic SSL with Let's Encrypt:
    
    - Automatic certificate provisioning
    - Auto-renewal before expiration
    - Custom certificate resolvers
    - Wildcard certificate support
  </Step>
  
  <Step title="Routing Rules">
    Advanced Traefik routing:
    
    ```yaml
    # Automatic generation
    http:
      routers:
        app-router:
          rule: "Host(`app.example.com`)"
          service: "app-service"
          tls:
            certResolver: "letsencrypt"
    ```
  </Step>
</Steps>

### Advanced Routing

<Accordion title="Path-Based Routing">
  Route different paths to different services:
  
  ```typescript
  [
    { host: "api.example.com", path: "/v1", port: 3000 },
    { host: "api.example.com", path: "/v2", port: 4000 }
  ]
  ```
</Accordion>

<Accordion title="Redirects">
  Configure HTTP to HTTPS and custom redirects:
  
  ```typescript
  {
    type: "www-to-non-www",
    target: "https://example.com"
  }
  ```
</Accordion>

<Accordion title="Load Balancing">
  Distribute traffic across multiple containers:
  
  ```yaml
  services:
    app-service:
      loadBalancer:
        servers:
          - url: "http://app-1:3000"
          - url: "http://app-2:3000"
        passHostHeader: true
  ```
</Accordion>

## Multi-Node & Docker Swarm

Scale applications across multiple servers for high availability.

### Docker Swarm Integration

<CardGroup cols={2}>
  <Card title="Multi-Server Setup" icon="server">
    Connect multiple servers to form a cluster:
    
    - Manager and worker nodes
    - Automatic service distribution
    - Built-in load balancing
    - Failure recovery
  </Card>
  
  <Card title="Service Replication" icon="copy">
    Run multiple replicas of your application:
    
    ```typescript
    serviceModeSwarm: {
      mode: "replicated",
      replicas: 3
    }
    ```
  </Card>
  
  <Card title="Placement Constraints" icon="filter">
    Control where services run:
    
    ```typescript
    placementSwarm: {
      constraints: [
        "node.role==worker",
        "node.labels.region==us-west"
      ]
    }
    ```
  </Card>
  
  <Card title="Update Strategies" icon="arrows-rotate">
    Zero-downtime rolling updates:
    
    ```typescript
    updateConfigSwarm: {
      parallelism: 1,
      delay: "10s",
      order: "start-first"
    }
    ```
  </Card>
</CardGroup>

### Swarm Configuration

```typescript
// Complete swarm configuration example
{
  serviceModeSwarm: {
    mode: "replicated",
    replicas: 3
  },
  placementSwarm: {
    constraints: ["node.role==worker"],
    preferences: [{ spread: "node.id" }]
  },
  updateConfigSwarm: {
    parallelism: 1,
    delay: "10s",
    failureAction: "rollback",
    order: "start-first"
  },
  restartPolicySwarm: {
    condition: "on-failure",
    delay: "5s",
    maxAttempts: 3,
    window: "120s"
  }
}
```
