## Create Application

Create a new application in a specific environment. This endpoint initializes an application with basic configuration that can be further customized with source providers and build settings.

### Endpoint

```
POST /application.create
```

### Request Body

<ParamField path="name" type="string" required>
  Display name for the application
</ParamField>

<ParamField path="environmentId" type="string" required>
  ID of the environment where the application will be created
</ParamField>

<ParamField path="appName" type="string">
  Internal application name used for Docker containers and routing. If not provided, will be auto-generated from the display name.
</ParamField>

<ParamField path="description" type="string">
  Optional description of the application
</ParamField>

<ParamField path="serverId" type="string">
  Server ID where the application will be deployed (required for cloud deployments)
</ParamField>

### Response

Returns the created application object with generated `applicationId` and default configuration.

<ResponseField name="applicationId" type="string">
  Unique identifier for the created application
</ResponseField>

<ResponseField name="name" type="string">
  Display name of the application
</ResponseField>

<ResponseField name="appName" type="string">
  Internal application name
</ResponseField>

<ResponseField name="environmentId" type="string">
  ID of the parent environment
</ResponseField>

<ResponseField name="applicationStatus" type="string">
  Current status: `idle`, `running`, `done`, or `error`
</ResponseField>

<ResponseField name="sourceType" type="string">
  Default source type (initially `github`)
</ResponseField>

<ResponseField name="buildType" type="string">
  Default build type (initially `nixpacks`)
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of creation
</ResponseField>

### Example Request

```bash
curl -X POST https://your-dokploy-instance.com/api/application.create \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Web App",
    "appName": "my-web-app",
    "description": "Production web application",
    "environmentId": "env_abc123",
    "serverId": "server_xyz789"
  }'
```

### Example Response

```json
{
  "applicationId": "app_def456",
  "name": "My Web App",
  "appName": "my-web-app",
  "description": "Production web application",
  "environmentId": "env_abc123",
  "serverId": "server_xyz789",
  "applicationStatus": "idle",
  "sourceType": "github",
  "buildType": "nixpacks",
  "createdAt": "2024-03-06T10:30:00.000Z",
  "env": null,
  "buildArgs": null,
  "memoryLimit": null,
  "cpuLimit": null
}
```

## Get Application

Retrieve details of a specific application including its configuration, deployment status, and Git provider access status.

### Endpoint

```
GET /application.one
```

### Query Parameters

<ParamField query="applicationId" type="string" required>
  ID of the application to retrieve
</ParamField>

### Response

Returns the complete application object with all configuration details.

<ResponseField name="applicationId" type="string">
  Unique identifier for the application
</ResponseField>

<ResponseField name="name" type="string">
  Display name of the application
</ResponseField>

<ResponseField name="appName" type="string">
  Internal application name
</ResponseField>

<ResponseField name="applicationStatus" type="string">
  Current status: `idle`, `running`, `done`, or `error`
</ResponseField>

<ResponseField name="sourceType" type="string">
  Source provider: `github`, `gitlab`, `bitbucket`, `gitea`, `docker`, `git`, or `drop`
</ResponseField>

<ResponseField name="buildType" type="string">
  Build system: `dockerfile`, `nixpacks`, `heroku_buildpacks`, `paketo_buildpacks`, `railpack`, or `static`
</ResponseField>

<ResponseField name="repository" type="string">
  Repository name (for Git-based sources)
</ResponseField>

<ResponseField name="owner" type="string">
  Repository owner or organization
</ResponseField>

<ResponseField name="branch" type="string">
  Git branch to deploy from
</ResponseField>

<ResponseField name="buildPath" type="string">
  Path within repository to build from
</ResponseField>

<ResponseField name="dockerfile" type="string">
  Path to Dockerfile (for dockerfile build type)
</ResponseField>

<ResponseField name="dockerImage" type="string">
  Docker image name (for docker source type)
</ResponseField>

<ResponseField name="env" type="string">
  Environment variables as key=value pairs
</ResponseField>

<ResponseField name="buildArgs" type="string">
  Build-time arguments
</ResponseField>

<ResponseField name="memoryLimit" type="string">
  Memory limit (e.g., "512m", "1g")
</ResponseField>

<ResponseField name="memoryReservation" type="string">
  Memory reservation
</ResponseField>

<ResponseField name="cpuLimit" type="string">
  CPU limit
</ResponseField>

<ResponseField name="cpuReservation" type="string">
  CPU reservation
</ResponseField>

<ResponseField name="hasGitProviderAccess" type="boolean">
  Whether the current user has access to the configured Git provider
</ResponseField>

<ResponseField name="unauthorizedProvider" type="string">
  Git provider name if access is not available
</ResponseField>

<ResponseField name="environment" type="object">
  Parent environment details
</ResponseField>

<ResponseField name="deployments" type="array">
  Array of deployment records
</ResponseField>

### Example Request

```bash
curl -X GET "https://your-dokploy-instance.com/api/application.one?applicationId=app_def456" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

## Update Application

Update application configuration including name, description, resource limits, and various deployment settings.

### Endpoint

```
POST /application.update
```

### Request Body

<ParamField path="applicationId" type="string" required>
  ID of the application to update
</ParamField>

<ParamField path="name" type="string">
  New display name
</ParamField>

<ParamField path="appName" type="string">
  New internal name
</ParamField>

<ParamField path="description" type="string">
  Updated description
</ParamField>

<ParamField path="env" type="string">
  Environment variables (newline-separated KEY=VALUE pairs)
</ParamField>

<ParamField path="memoryLimit" type="string">
  Memory limit (e.g., "512m", "2g")
</ParamField>

<ParamField path="memoryReservation" type="string">
  Memory reservation
</ParamField>

<ParamField path="cpuLimit" type="string">
  CPU limit (e.g., "0.5", "2")
</ParamField>

<ParamField path="cpuReservation" type="string">
  CPU reservation
</ParamField>

<ParamField path="command" type="string">
  Custom command to run
</ParamField>

<ParamField path="args" type="array">
  Command arguments
</ParamField>

<ParamField path="autoDeploy" type="boolean">
  Enable automatic deployments on Git push
</ParamField>

<ParamField path="cleanCache" type="boolean">
  Clear build cache on next deployment
</ParamField>

<ParamField path="replicas" type="number">
  Number of replicas (for Swarm mode)
</ParamField>

### Example Request

```bash
curl -X POST https://your-dokploy-instance.com/api/application.update \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "applicationId": "app_def456",
    "name": "My Updated Web App",
    "description": "Updated production application",
    "memoryLimit": "2g",
    "cpuLimit": "1.5",
    "autoDeploy": true
  }'
```

### Example Response

```json
{
  "success": true
}
```

## Search Applications

Search and filter applications across your organization with pagination support.

### Endpoint

```
GET /application.search
```

### Query Parameters

<ParamField query="q" type="string">
  Search term to match against name, appName, description, repository, owner, or dockerImage
</ParamField>

<ParamField query="name" type="string">
  Filter by application name
</ParamField>

<ParamField query="appName" type="string">
  Filter by internal app name
</ParamField>

<ParamField query="description" type="string">
  Filter by description
</ParamField>

<ParamField query="repository" type="string">
  Filter by repository name
</ParamField>

<ParamField query="owner" type="string">
  Filter by repository owner
</ParamField>

<ParamField query="dockerImage" type="string">
  Filter by Docker image name
</ParamField>

<ParamField query="projectId" type="string">
  Filter by project ID
</ParamField>

<ParamField query="environmentId" type="string">
  Filter by environment ID
</ParamField>

<ParamField query="limit" type="number" default="20">
  Maximum number of results (1-100)
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of results to skip
</ParamField>

### Response

<ResponseField name="items" type="array">
  Array of matching applications
</ResponseField>

<ResponseField name="total" type="number">
  Total number of matching applications
</ResponseField>

### Example Request

```bash
curl -X GET "https://your-dokploy-instance.com/api/application.search?q=web&limit=10&offset=0" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```
